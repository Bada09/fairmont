01 / ACCESS

Acesso e configuração de perfil

Primeiro bloco operacional para garantir que o usuário entre com o contexto correto antes de iniciar qualquer simulação.

module.access_profilerequired
1

Abrir o aplicativo

Localize o ícone da Rhapsody no PC ou smartphone. Ao entrar, o sistema reconhece o perfil profissional e carrega os cenários liberados para sua função.

2

Validar o contexto do usuário

Confira se os use cases exibidos correspondem ao seu cargo, área ou trilha de aprendizagem. Caso a lista pareça incorreta, registre o caso com o gestor ou suporte.

3

Ajustar voz e idioma

Toque na foto no canto superior direito para acessar o perfil. Selecione a voz da IA e o idioma padrão de operação do aplicativo.

Critério de pronto

O usuário está pronto para treinar quando o catálogo aparece com cenários relevantes, o idioma está correto e a voz selecionada é adequada ao exercício.

03 / USE_CASES

Encontrar e selecionar cenários

O catálogo funciona como biblioteca de situações reais de trabalho, filtrável por objetivo de desenvolvimento.

module.catalog_filterscompetence + kpi
A

Todos os use cases

Exibe a lista completa de cenários disponíveis para o perfil reconhecido. Útil para exploração inicial ou treinamentos livres.

B

Por competências

Filtra por habilidades comportamentais ou comerciais, como negociação, escuta ativa, argumentação, gestão de conflito e feedback.

C

Por KPIs

Relaciona o treino a indicadores de performance, como taxa de transformação, retenção, qualidade de atendimento ou conversão.

04 / RUN

Executar uma simulação

O Run é a sessão prática. O usuário fala, a IA responde e a conversa evolui como role-play orientado por cenário.

module.run_sessionwalkie-talkie mode
SelecionarToque no use case desejado e leia o contexto.
ConfigurarEscolha papel, voz e idioma da sessão.
IniciarToque em “Passar ao run”.
ConversarPressione para falar e pressione novamente ao terminar.
EncerrarFinalize quando o objetivo pedagógico tiver sido explorado.
Modo walkie-talkie

Toque uma vez para começar a falar. Toque novamente quando terminar. A IA responde em seguida, alternando os turnos da conversa.

Uso em grupo

O mesmo mecanismo permite exercícios coletivos, com uma pessoa ou um grupo respondendo à IA durante o role-play.

Ponto de atenção

Antes de iniciar, leia o objetivo pedagógico. Ele define o comportamento que será treinado e melhora a qualidade do debriefing.

05 / DEBRIEFING

Resultados, feedback e evidências

Após o Run, a plataforma organiza o aprendizado em formatos complementares para análise individual ou compartilhada.

module.run_resultspost-session
Debriefing em áudio

A IA apresenta verbalmente pontos fortes, oportunidades de melhoria e recomendações práticas para a próxima tentativa.

Nota

A avaliação numérica é calculada com base nos critérios do exercício, mas só é gerada se o usuário solicitar.

Script da conversa

Transcrição completa dos turnos da simulação, útil para revisar linguagem, objeções, argumentos e momentos de hesitação.

Debriefing em texto

Versão escrita do feedback, adequada para estudo posterior, registro e comparação entre tentativas.

Autoavaliação

O usuário avalia o próprio run, ajudando a qualificar cenários relevantes e consolidar metacognição.

Leitura recomendada

Combine três sinais: o que a IA apontou, o que aparece no script e o que o próprio usuário percebeu. Essa triangulação reduz leitura apressada de desempenho.

06 / GOVERNANCE

Compartilhamento e melhoria contínua

A Rhapsody não termina no feedback. O valor aumenta quando o usuário repete, comenta e compartilha com intenção clara.

module.collaborationuser-controlled
AçãoComo funcionaBoa prática
CompartilharEnvie um run para gestor, colega ou equipe. O compartilhamento é uma escolha do usuário.Compartilhe com contexto: objetivo do treino, dúvida principal e ponto que deseja discutir.
RelançarRepita o mesmo use case sempre que quiser.Faça uma nova tentativa aplicando uma recomendação específica do debriefing anterior.
ComentarEnvie sugestões de melhoria para o cenário, como “o personagem poderia ser mais agressivo”.Descreva comportamento esperado, intensidade e impacto no aprendizado.
learning_looprepeatable
loop:
  1. praticar o cenário
  2. revisar debriefing e script
  3. escolher um comportamento para melhorar
  4. relançar o use case
  5. comparar a nova tentativa
  6. compartilhar apenas quando fizer sentido para o desenvolvimento
07 / SECURITY

Segurança, privacidade e LGPD

Bloco para orientar uso responsável, especialmente em contextos corporativos e treinamentos com dados sensíveis.

module.privacy_baselinelgpd-aware
S

Acesso controlado

Use somente credenciais autorizadas e não compartilhe sessões com pessoas fora do público previsto.

P

Dados pessoais

Evite inserir dados reais de clientes, como documentos, telefones, endereços ou informações sensíveis.

G

Governança

Runs compartilhados devem ter finalidade de aprendizagem, com consentimento e contexto claro para quem recebe.

Orientação de conformidade

Ao usar exemplos reais, anonimize nomes, números e referências. Prefira descrever a situação de forma genérica e preservar detalhes que identifiquem clientes ou colaboradores.

Uso ético do feedback

O debriefing deve apoiar desenvolvimento. Evite transformar uma simulação isolada em julgamento definitivo de competência.
